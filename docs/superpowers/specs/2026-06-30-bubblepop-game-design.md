ווידג'ט משחק קטן בדשבורד המערכת. בועות עולות מתחתית קנבס — כל בועה מייצגת "פנייה נכנסת" מסוג שונה. הנציג לוחץ לפוצץ אותן לפני שמגיעות לראש. ניקוד מצטבר all-time עם לידרבורד פנימי top-5.

**מטרה:** עידוד קל ובלתי מחייב — ממכר אך לא מפריע לעבודה.

| `public/api/game/score.php` | POST — שמירת ניקוד |
| `public/api/game/leaderboard.php` | GET — top-5 + ניקוד אישי |

### DB

טבלה `game_scores` (כבר נוצרה):
```sql
CREATE TABLE game_scores (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  score INT NOT NULL DEFAULT 0,
  played_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_user (user_id)
);
```
שורה אחת לכל משתמש. `INSERT ... ON DUPLICATE KEY UPDATE` — מתעדכן רק אם הניקוד גבוה מהקיים.

**בועות נדירות:** גדולות יותר (55–65px), אנימציית ניצנוץ (glow pulsing).

### פיזיקה

- גודל בועה: 40–55px רגיל, 55–65px נדיר
- מהירות התחלתית: 0.8px/frame
- עלייה במהירות: +0.05px כל 10 שניות
- מספר בועות בו-זמנית: מתחיל 3, +1 כל 15 שניות (מקסימום 8)
- בועה חדשה מתווספת כשבועה קיימת נפוצה או עוברת את הראש

### ניקוד

- לחיצה על בועה: +ניקוד לפי טבלה
- בועה שעוברת את הראש: -5 (לא יורד מתחת לאפס)
- כל 50 נקודות: flash קצר על הניקוד ("Level!")

### שמירה

- אוטומטית כל 30 שניות אם הניקוד השתפר
- בעת `visibilitychange` / `beforeunload`
- API מקבל רק אם `new_score > existing_score`

## API Endpoints

### POST `/api/game/score`

**Headers:** `X-CSRF-TOKEN`  
**Body:** `{ "score": 350 }`  
**Auth:** session חובה  
**לוגיקה:** `INSERT ... ON DUPLICATE KEY UPDATE score=IF(score < VALUES(score), VALUES(score), score)`  
**Response:** `{ "ok": true, "saved": true/false }`

### GET `/api/game/leaderboard`

**Auth:** session חובה  
**Response:**
```json
{
  "top5": [
    { "name": "דני כהן", "score": 890, "is_me": false },
    ...
  ],
  "my_score": 240
}
```

---

## אבטחה

- שני endpoints דורשים session פעיל
- POST מוודא CSRF token
- ניקוד מוגבל בצד שרת: `score > 0 AND score < 99999`
- rate limit פשוט: לא מקבלים יותר מ-10 בקשות POST לדקה למשתמש (נבדוק בAPI)
